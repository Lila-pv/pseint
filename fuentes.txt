
 Inicio Rapido
===============

Si solo quiere compilar y utilizar instale las dependencias (ver requisitos) y 
el programa utilice el comando:
  ming32-make -f Makefile.w32
si esta en Microsoft Windows, o el comando:
  ming32-make -f Makefile.lnx 
si esta en GNU/Linux o Mac Os. Los binarios se ubicarán en el directorio bin. 
Desde allí puede ejecutar wxPSeInt para comenzar a utilizar el programa.



 Requisitos para compilar PSeInt
=================================

Para compilar pseint se requieren las bibliotecas wxWidgets-2.8, zlib, libpng, libjpeg, y 
glut (con sus dependencias). En la mayoría de los sistemas GNU/Linux estas se pueden 
conseguir facilmente a través del repositorio de la distribución (aunque usualmente 
la versión disponible de wx es Unicode mientras que la que utilizo en mis compilaciones 
es ANSI, se requiere editar las variables LIBS Y CFLAGS en el archivo wxPSeInt/Makefile.lnx
para utilizar la version Unicode). Si utiliza Windows puede descargar el IDE ZinjaI que 
incluye el compilador gcc y la biblioteca wxWidgets en su versión correcta. Solo hace falta 
agregar las bibliotecas zlib, libpng y libjpeg que puede descargar desde 
gnuwin32.sourceforge.net. Para ello debe descargar "development files" y descomprimir en una 
nueva carpeta misc dentro de mingw para compilar, y "binaries" y copiar los .dll a la carpeta 
de pseint para ejecutar, o dist para usar los scripts que generan el instalador desde 
GNU/Linux mediante wine.



 Acerca de los fuentes de PSeInt
=================================

Contenido de los directorios:
  pseint: fuentes del interprete (verifica sintaxis e interpreta un algoritmo)
  psdraw: fuentes del graficador (genera y muestra o graba diagramas de flujo)
  psdraw2: fuentes del editor de diagramas de flujo
  psexport: fuentes del exportador (convierte de pseudocodigo a codigo C++)
  wxPSeInt: fuentes de la nueva interfaz (editor)
  psterm: fuentes de la terminal que se utiliza para ejecutar desde wxPSeInt
  updatem: fuentes del programa que busca actualizaciones
  test: pseudocódigos con casos de prueba y scripts para correrlos de forma automática
  bin: demas archivos necesarios para ejecutar wxPSeInt (imagenes, documentacion, ayuda, etc).
  dist: archivos adicionales para generar paquetes e instaladores

Adentro de cada directorio hay tres Makefiles:
  Makefile.lnx para compilar las versiones para GNU/Linux
  Makefile.w32 para compilar las versiones para Windows desde Windows, con MinGW
  Makefile.wine para compilar las versiones para Windows desde GNU/Linux, con Wine+MinGW
  Makefile.m32 para compilar las versiones para Mac OS X i686 desde GNU/Linux, con IMCROSS
  Makefile.ppc para compilar las versiones para Mac OS X powerpc desde GNU/Linux, con IMCROSS
Para compilar en o para Windows puede que necesite retocar las variables 
definidas en mingw.conf. Los archivos intermedios de la compilación se ubican en temp.
Además, hay un archivo Makefile.pack que sirve para generar los paquetes e instaladores
de todo el software (solo desde GNU/Linux, y requiere wine+mingw+wxwidgets+nsis)

Los fuentes son exactamente los mismos tanto para GNU/Linux como para Windows.
Los paquetes que se distribuyen desde el sitio oficial se construyen con gcc y en
GNU/Linux (Ubuntu 9.10), aún la versión para de Windows (utilizando mingw, wine y 
Slackware 13).
El instalador para Windows se construye con NSIS (nsis.sourceforge.net). El 
framework utilizado para el desarrollo de la GUI es wxWidgets (www.wxwidgets.org)
El lenguaje empleado en todos los fuentes es C++.

En cada directorio de fuentes hay un archivo .zpr. Este es el archivo de proyecto.
Se abren con un IDE llamado ZinjaI (disponible en http://zinjai.sourceforge.net).
